方案 B: 新会话空状态引导

新建会话默认全部关闭。先打开你需要的 agent，再开始发送消息。

发送消息时
若没有启用 agent，后端返回明确提示，前端展示空态引导，不直接发往任何 agent。
关闭当前 agent 后
本轮状态先保留，下一条消息起不再沿用它作为 currentAgent。
R

推荐方向

聊天页顶部直接放 session 内 agent 开关，并配合新会话空状态引导。信息密度高一点，但切换最快，也最贴近用户发消息前的决策点。

S

更分离的方向

把开关放进单独的“会话设置”抽屉，聊天顶部只显示已启用 agent。页面更干净，但多一次进入设置的操作。
